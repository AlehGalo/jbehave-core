Cross Reference

JBehave can generate a JSON and XML cross reference at the end of a run.  Details of Stories and Scenarios are in the document, as are details of step matches.

The intended use of these is with webapps, and we have one to start with called Story Navigator (link TBA)

Setup

public class YourStories extends JUnitStories {
    private CrossReference crossReference = new CrossReference();
    @Override
    public Configuration configuration() {
    return new SeleniumConfiguration()
            .useStepMonitor(crossReference.getStepMonitor()))
            .useStoryReporterBuilder(
                    new StoryReporterBuilder()
                            .withCrossReference(crossReference));
    }
}

We are hiding other methods in the above code fragment to make it clear just what you have to do for setting up the cross reference.  In particular:
  • you are likely to be chaining more than one step monitor together in the useStepMonitor(..) line
  • your StoryReporter has more  configuration typically
  • there are other methods to implement in JUnitStories for finding steps and stories.
Real example: See EtsyStories in the tutorial.
